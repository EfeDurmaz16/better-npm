01_ VISION

Better turns node_modules from a black box into an observable system.

Every install, every package, every MB, and every vulnerability becomes measurable.

The toolkit is designed for long-lived codebases, monorepos, and infrastructure teams that need to reason about dependency risk and cost, not just install faster.

Non-invasive

Wrap npm, pnpm, Yarn. Do not replace them.

Measurable

Every run emits metrics: time, size, cache hits.

Explainable

Every score and warning includes concrete reasons.

Observable

NDJSON logs, JSON APIs, local dashboard for CI/SRE.

02_ WHAT BETTER DOES

better install

Wraps your package manager, measures time and size, and makes installs auditable.

> better install --json
{
  "durationMs": 73241,
  "packagesInstalled": 154,
  "sizeAddedBytes": 524288000,
  "cacheHits": 87,
  "cacheMisses": 9
}

better analyze

Scans node_modules, builds dependency graph, computes logical and physical size.

> better analyze
Total packages: 523
Direct dependencies: 24
Logical size: 1560 MB
Physical size: 980 MB
Max depth: 12
Duplicates: 37

better doctor

Runs health checks and returns a 0-100 score with itemized findings and actions.

> better doctor
Health Score: 73/100

ERRORS:
- request is deprecated, replace with node-fetch
- lodash@4.17.20 has known vulnerabilities

WARNINGS:
- debug has 3 versions installed

better cache

Shared content-addressable cache across projects, with stats and garbage collection.

> better cache stats
Entries: 18,452
Size: 186.4 GB
Hit ratio: 0.84
Oldest: 2025-12-01T08:33:12Z

04_ FOR TEAMS

Infrastructure & Platform Teams

Benchmark install performance across repos, enforce health thresholds in CI, and keep shared cache usage under control instead of letting node_modules sprawl.

Security & Compliance

Track deprecated and vulnerable packages, integrate findings into existing security pipelines, and prove dependency risk is being measured continuously.

Product & Engineering Leaders

Quantify dependency cost in time and disk, identify monorepo hotspots, and justify refactors with hard numbers instead of anecdotes.

05_ ROADMAP

# Q1: Shared cache + install metrics
# Q2: Full analyzer, health score, local dashboard
# Q3: Advanced cache intelligence, per-team policy engine
# Q4: Multi-language dependency toolkit experiments

Better starts with Node.js and is intentionally designed as a dependency observability layer that can extend to other ecosystems. Long term, dependency risk, cost, and health become as trackable as latency and error rates.
